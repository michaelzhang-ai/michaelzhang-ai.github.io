SGLang CI Infrastructure & Performance Benchmarking on AMD GPUs

AMD · Sep 2024 — Present

MI300X · MI325X · MI355X

Sibo Zhang — Member of Technical Staff, AMD

Overview

Primary developer of an in-house CI toolkit and performance benchmarking infrastructure for SGLang on AMD Instinct MI300X/MI325X/MI355X GPUs. The system covers nightly accuracy validation (GSM8K, MMMU), throughput & latency benchmarks, unit tests across ROCm 7.0/7.2, and model enablement for new LLM/VLM architectures — all orchestrated through Docker-based self-hosted GitHub Actions runners with a Flask-based monitoring dashboard.

Key Contributions

  • CI Infrastructure. Built nightly accuracy (GSM8K, MMMU), performance benchmarks, and unit tests across ROCm 7.0/7.2 with Docker-based self-hosted GitHub Actions runners.
  • CI Dashboard. Built Flask web dashboard with SQLite backend for CI health monitoring, performance trend visualization, AMD vs NVIDIA upstream test coverage tracking, and PR-level test results.
  • Model Enablement. Enabled 6+ LLM/VLM architectures on AMD: MiniMax-M2.5 (256-expert MoE, 95% GSM8K), Qwen 3.5, GLM-5 (NSA), Kimi-K2.5 (MLA+MoE), DeepSeek-OCR-2, Z-Image-Turbo. End-to-end: architecture research, backend selection (aiter/triton), test authoring, CI registration.
  • Upstream Contributions. 16 PRs to sgl-project/sglang (10 merged), 20 PRs to ROCm/sglang-ci (18 merged). Fixed AMD-specific regressions in aiter backend.
  • Performance. Benchmarked CLIP and Janus-Pro-7B on MI300X vs H100, achieving 1.30x and 1.20x geometric mean speedup. Throughput/latency regression testing for DeepSeek-V3, Grok.
